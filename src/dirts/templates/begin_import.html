Import requirements

This utility helps in uploading and importing into our Echelon database a list of DIRTs from files. In order to succeed with the import, the file must meet the following criteria:

  • The file must be a comma separated values file (CSV)
  • The first row must have the following columns in this order:
    1. Date Created
    2. Status
    3. Priority
    4. Version
    5. Reference
    6. Description
    7. Comments
    8. Date Closed
  • Data rows columns must match with first row columns

Formatting criteria applies to each column, described below:

  • Date Created - (dd/mm/yyyy). Must not be empty.
  • Status - ('Open', 'Closed'). Must not be empty.
  • Priority - ('High', 'Medium', 'Low', 'Observation'). Must not be empty.
  • Version - Software or hardware version number. Must not be empty.
  • Reference - One line summary of DIRT. Must not be empty.
  • Description - Body of DIRT. Must not be empty.
  • Comments - Additional information of DIRT. Optional.
  • Date Closed - (dd/mm/yy). Optional.
